# Security Policy

## Supported Versions

Security fixes target the latest tagged release.

## Reporting a Vulnerability

Please report vulnerabilities privately to Florin Badita at florin.badita@gmail.com.

Do not open public issues for suspected vulnerabilities involving anonymity, proof verification, credential handling, or data leakage.

## Baseline

- No secrets belong in git.
- No secrets belong in frontend bundles.
- Attendee credentials are generated and distributed out of band.
- Published votes must be validated before they are counted.
