# 0004 - Static Data Contract

## Status

Accepted

## Context

Mode A has no scheduled data backend. The app still needs a stable contract for static assets and shareable room data.

## Decision

The static asset contract is:

- `docs/index.html` and hashed assets generated by Vite.
- `docs/404.html` copied from `index.html` for SPA fallback.
- WASM and worker assets are loaded from the Vite asset graph or package CDN-compatible URLs.

The room contract is encoded in the URL hash as a compressed JSON payload:

- `schemaVersion`
- `roomId`
- `title`
- `createdAt`
- `polls`
- `attendeeCommitments`
- `proofProfile`

The live state contract is a Yjs document containing votes, questions, room metadata, and append-only event records.

## Consequences

- A room can be shared without a server.
- The public page can host many independent rooms.
- URL payload size limits are real; very large attendee groups should use imported room files in a later version.

## Alternatives Considered

- Static JSON in `docs/data/`: rejected for v1 because rooms are user-created, not repository-curated.
- Runtime API contract: rejected by ADR 0001.
