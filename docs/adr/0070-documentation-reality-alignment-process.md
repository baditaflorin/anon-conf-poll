# 0070 - Documentation-Reality Alignment Process

## Status

Accepted.

## Context

The README made a few claims that were true in spirit but too broad for the shipped static app.

## Decision

README claims must map to tests or documented limitations. Phase 3 updates the README with:

- verified feature checklist
- honest WebRTC/static-mode limitations
- state import/export guidance
- no service-worker offline guarantee

## Consequences

The live page remains ambitious, but the repo does not overpromise.

## Alternatives Considered

Keeping aspirational text was rejected because Phase 3 is about completeness, not hype.
